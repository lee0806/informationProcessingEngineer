
### OSI 7계층

> 물 데 네 전 세 표 응
> 물리 (비트) - 데이터링크 (프레임) - 네트워크 (패킷) - 전송 (세그먼트) - 세션 (대화 관리) - 표현 (암호화) - 응용 (HTTP, FTP)

### 결합도 (Coupling)

> 내 공 외 제 스 자
> 내용 > 공통 > 외부 > 제어 > 스탬프 > 자료
> 자료 결합도 높음 -> 낮음
> 결합도는 낮을수록 좋음

### 응집도 (Cohesion)

> 우 논 시 절 통 순 기
> 우연적 < 논리적 < 시간적 < 절차적 < 통신적 < 순차적 < 기능적
> 응집도가 낮은것에서 높은 순

### 소프트웨어 생명주기 (SDLC)

> 타 분 설 구 테 유
> 타당성 -> 분석 -> 설계 -> 구현 -> 테스트 -> 유지보수

### 요구사항 분석 과정

> 도 개 명 확
> 도출 -> 분석 ->명세 -> 확인

### 테스트 단계

> 단 통 시 인 유
> 단위 테스트 -> 통합 테스트 -> 시스템 테스트 -> 인수 테스트 -> 유지보수 테스트

### 테스트 기법 (White)

> 문 결 조 조 기 데
> 문장, 결정, 조건, 조합, 기본경로, 데이터 흐름

### 테스트 기법 (Black)


> 동 경 결 상 원 유 분
> 동등분할, 경계값, 결정표, 상태전이, 원인결과, 유스케이스, 분류 트리

### 데이터베이스 3정규형

> 원 부 이 결 다 조
> 원자값 (1NF), 부분함수종속제거 (2NF), 이행함수종속제거 (3NF), 결정자가 후보키가 아닌 함수 종속 제거 (BCNF 보이스-코드 정규형), 다치 종속 제거 (4NF), 조인 종속 제거 (5NF)

### DB 트랜잭션 특성

> ACID
> 원자성, 일관성, 고립성, 지속성

### DB 무결성

> 개 참 도 키
> 개체 무결성, 참조 무결성, 도메인 무결성, 키 무결성

### DB 제약조건

> 프 외 체 기 도 널
> Primary / Foreign / Check / Unique / Default / Not Null

### 데이터모델 구성요소

> 개 속 관
> 개체, 속성, 관계

### 스케줄링 알고리즘

> FCFS SJF HRN RR MLQ MLFQ
> 선입선출 → 단기작업우선 → HRN → 라운드로빈 → 다단계큐


### 디자인 패턴 분류

> 생 구 행
> 생성, 구조, 행위 패턴

> 생성 패턴 (객체를 어떻게 만들까?)
> 싱 팩 추 빌 프
> 싱글톤, 팩토리 메서드, 추상 팩토리, 빌더, 프로토타입

> 구조 패턴 (클래스나 객체를 어떻게 연결할까?)
> 어 브 컴 데 퍼 플 프
> 어뎁터, 브릿지, 컴포지트, 데코레이터, 퍼사드, 플라이웨이트, 프록시

> 행위 패턴 (객체들이 어떻게 상호작용할까?)
> 체 커 템 전 옵 상 비 미 메 인 이
> 체인 오브 리스폰시빌리티, 커맨드, 템플릿 메서드, 전략, 옵저버, 상태, 비지터, 미디에이터, 메멘토, 인터프리터, 이터레이터

### 보안 3요소

> 기 무 가
> 기밀성, 무결성, 가용성

### 소프트웨어 아키텍처 4 + 1 뷰

> 논 프 배 유 시
> 논리 / 프로세스 / 배포 / 유스케이스 / 시나리오 뷰

### 요구공학 프로세스

> 도 분 명 검 관
> 도출 - 분석 - 명세 - 검증 - 관리
