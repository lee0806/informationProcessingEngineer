## 프로젝트 준비 과정

"김아무개"라는 친구는 개발자로 취업하게 되었습니다.
처음 회사에 들어갔을 때 **1. 응용 SW 기초 활용과 2. 네트워크의 기초**를 교육 받게 됩니다.
교육이 끝나고 개발 프로젝트에 투입하게 되면 클라이언트의 **3. 요구사항을 확인** 하게 됩니다.
요구사항을 기반으로 **4. 화면 설계**를 진행하고
데이터의 구조를 조직화하고 설계하기 위해서 **5. 데이터의 입출력을 구현**하고 서로 다른 시스템간의 원할한 통신을 위한 연계 모듈을 위해 **6. 통합 구현**을 합니다.

1. 응용 SW 기초 활용
	1. 윈도우 / 유닉스의 운영체제 특징
	2. 메모리 배치와 관리 방법
	3. 프로세스 관리와 스케줄링 (선점 / 비선점)
	4. 클라우드 서비스 유형
2. 네트워크의 기초 활용
	1. OSI 7 Layer의 특징
	2. 프로토콜의 3요소
	3. IPv4 / IPv6
	4. 라우팅 프로토콜 (3계층 - RIP / OSPF / BGP / 라우팅 알고리즘)
	5. 전송 계층 (4계층 - TCP의 특징)

3. 요구사항 확인
	1. 소프트웨어 개발 방법론
		1. 소프트웨어 생명주기 모델 종류 (폭포수, 나선형 등)
		2. 나선형 모델의 절차
		3. 소프트웨어 개발방법론의 종류 (애자일, 객체지향 방법론 등)
		4. 비용산정모형 분류 (하향식(델파이) / 상향식 (COCOMO) 선정방법)
		5. 일정관리 모델 종류
	2. 현행 시스템 분석
		1. 소프트웨어 아키텍처 4 + 1 뷰 / 패턴 유형 / 비용 평가 모델의 종류
		2. 디자인 패턴 종류
		3. OSI 7계층
	3. 요구사항 확인
		1. 요구사항 개발 프로세스
		2. 요구사항 도출 단계 주요 기법 (델파이 기법, 설문조사, 브레인스토밍)
		3. 정형 기술 검토 (동료검토, 워크스루, 인스펙션)

4. 화면 설계
	1. UI 요구사항 확인
		1. UI의 유형 (CLI, GUI, NUI, OUI)
		2. UI 설계 원칙 (직관성, 유효성 등)
		3. UI 설계 지침 (사용자 중심, 일관성)
		4. UI 요구사항 구분 (기능적 / 비기능적)
		5. UI 품질 요구 사항 (기능성, 신뢰성 등)
	2. UI 설계
		1. UML 특징 (가시화, 구축, 명세화, 문서화)
		2. UML 구성요소 (사물, 관계, 다이어그램)
		3. UML 다이어그램 (구조적 (클래스, 객체, 패키지), 행위적 (유스케이스, 시퀀스))
		4. UI 시나리오 문서의 작성 요건 (완전성, 일관성)

5. 데이터 입출력 구현
	1. 논리 데이터 저장소 확인
		1. 데이터 모델링 절차 (요구사항 분석 / 개념 / 논리 / 물리 / 구현)
		2. 일반 집합 연산 (합 / 교 / 차집합, 카티션 프로덕트)
		3. 논리데이터 모델링 속성 (개체, 속성, 관계)
		4. 이상현상 (삽입 / 삭제 / 갱신)
		5. 정규화 단계(도부이결다조), 반정규화
	2. 물리 데이터 저장소 설계
		1. 파티셔닝의 종류 (레인지 / 해시 / 리스트 / 컴포지트 파티셔닝)
		2. 파티션의 장점
	3. 데이터베이스의 기초 활용하기
		1. 데이터베이스의 유형 (키-값 DBMS 등)
		2. DBMS의 특징 (무결성, 일관성 등)
		3. 데이터마이닝 주요 기법 (분류 / 연관 / 연속 규칙, 데이터 군집화)

6. 통합 구현
	1. 연계 데이터 구성
		1. 연계 요구사항 분석 기법 및 참고 문서
		2. 인터페이스 명세서 구성요소
	2. 연계 매커니즘 구성
		1. 주요 연계 기술 (직접 (DB링크, JDBC 등) / 간접 (연계 솔루션 (EAI), 소켓 등) 연계)
	3. 내외부 연계 모듈 구현
		1. EAI 구축 유형
		2. ESB 방식

## 프로젝트 개발 작업

프로젝트 준비가 완료되면 프로젝트에 맞는 개발 환경을 구성 및 설계를 위해 **7. 서버 프로그램을 구현**합니다.
서버 프로그램에 맞추어 개발을 하기 위해선 개발 언어를 선택하고 언어의 특성 및 라이브러리를 기반으로 SW를 구현하기 위해 **8. 프로그래밍 언어를 활용**하도록 합니다. 또한 데이터를 정의하고 조작하며 제어할 수 있도록 **9. SQL을 응용**하도록 합니다.
SW와 연계 모듈간의 데이터 관계를 분석하고 효율적인 연계가 이루어지도록 **10. 인터페이스를 구현**합니다. 그리고 보안 취약점을 제거하고 해킹에 대비하기 위해 **11. 소프트웨어 개발 보안 설계**를 합니다


7. 서버 프로그램을 구현
	1. 개발환경 구축
		1. 개발 도구의 분류 (빌드 / 구현 / 테스트 / 형상관리 도구)
		2. 형상 관리 절차 (식별 / 통제 / 감사 / 기록)
		3. 소프트웨어 형상 관리 도구 유형 (공유 폴더 방식, 클라이언트 / 서버 방식, 분산 저장소 방식)
	2. 공통 모듈 구현
		1. 응집도
		2. 결합도
		3. MVC 패턴 역할
	3. 서버 프로그램 구현
		1. 서버 프로그램 세부 구현 프로세스 (DTO>DAO 등)
	4. 배치 프로그램 구현
		1. 배치 프로그램의 유형 (이벤트 / 온디맨드 / 정기 배치)
		2. Cron 표현식

8. 프로그래밍 언어 활용 (각 언어별 (C, 자바, 파이썬)로 출제됨)
	1. 기본 문법 활용
		1. 데이터 타입
		2. 기본 저장 공간
		3. 자료형
		4. 연산자
		5. 함수
		6. 명령문
		7. 클래스
		8. 인터페이스
	2. 언어 특성 활용
		1. 객체 구성 요소 (객체, 속성, 메서드)

9. SQL 응용
	1. 데이터베이스의 기본
		1. 트랜잭션의 특성 (원자성, 일관성 등)
		2. 트랜잭션의 상태
		3. TCL 명령어 (커밋, 롤백, 체크포인트)
		4. 병행제어 미보장 시 문제점 (갱신 손실 등)
		5. 병행 제어 기법 (로킹 등)
		6. 회복 기법 종류 (로그 기반 회복 기법 등)
		7. DDL 대상 (도메인, 스키마 등)
		8. 인덱스 종류 (순서 / 해시 인덱스 등)
		9. DDL 명령어 (CREATE, ALTER 등)
		10. DML 유형 (SELECT, INSERT 등)
		11. GRANT 명령어 (GRANT ON TO)
		12. REVOKE 명령어 (REVOKE ON FROM)
	2. 응용 SQL 활용
		1. 윈도 함수 분류 (순위 / 행 순서, 그룹 내 비율 함수)
	3. 응용 SQL 활용
		1. 윈도 함수 분류 (순위 / 행 순서, 그룹 내 비율 함수)
	4. 절차형 SQL 활용
		1. 절차형 SQL 종류 (프로시저, 사용자 정의 함수, 트리거)

10. 인터페이스 구현
	1. 인터페이스 설계 확인
		1. EAI 구축 유형
	2. 인터페이스 기능 구현
		1. JSON, XML, AJAX
		2. 데이터베이스 암호화 알고리즘 (대칭키 / 비대칭키 / 해시 암호화 알고리즘)
	3. 인터페이스 구현 검증
		1. 인터페이스 구현 검증 도구 (xUnit, FitNesse 등)
		2. 인터페이스 오류 처리 방법

11. 소프트웨어 개발 보안 설계
	1. SW 개발 보안 설계
		1. SW 보안의 3대 요소 (기밀성, 무결성, 가용성)
		2. SW 개발 보안 용어 (자산, 위협, 취약점, 위험)
		3. DDoS 공격 구성요소 (핸들러, 에이전트 등)
		4. 인증 기술 유형 (지식 / 소지 / 생체 / 특징 기반)
		5. 접근 통제 보호 모델 (벨라파둘라 / 비바)
		6. 서버 접근 통제 유형 (임의적 / 강제적 / 역할기반 접근 통제)
		7. 암호 알고리즘 상세 (대칭 / 비대칭 / 해시)
		8. IPSec
		9. 개인정보보호 관련 법령 (개인정보 보호법 등)
	2. 소프트웨어 개발 보안 구현
		1. 시큐어 코딩 가이드 적용 대상 (입력데이터 검증 및 표현, 보안 기능 등)
		2. 입력 데이터 검증 및 표현 취약점 (XSS, CSRF, SQL Injection)

## 프로젝트 마무리 작업

어느 정도 개발이 완료된 프로젝트를 클라이언트에게 제공하기 전에 요구사항대로 구현되었는지 검증하기 위해 **12 .애플리케이션 테스트 관리**를 진행하고 테스트를 통해 개발이 완료된 제품을 설치와 사용에 필요한 환경 등 전체적인 내용을 포함하는 메뉴얼을 작성하여 고객에게 전달하도록 **13. 제품 소프트웨어 패키징**을 하면 끝난다.

12. 애플리케이션 테스트 관리
	1. 애플리케이션 테스트 케이스 설계
		1. 소프트웨어 테스트 필요성 (오류 발견 / 오류 예방 / 품질 향상)
		2. 소프트웨어 테스트의 원리 (살충제 패러독스 등)
		3. 화이트 / 블랙 박스 유형, 테스트 목적에 따른 분류 (회복 / 안전 테스트 등)
		4. 성능 테스트 상세 유형 (부하 / 스트레스 테스트 등)
		5. 테스트 종류에 따른 분류 (명세 / 구조 / 경험 기반 테스트)
		6. 테스트 커버리지 유형 (기능 기반 / 라인 / 코드 커버리지)
		7. 경험기반 테스트 유형 (오류 추정, 체크리스트 등)
		8. 테스트 오라클 종류 (참 / 샘플링 / 휴리스틱 / 일관성 검사 오라클)
		9. 테스트 레벨 종류 (단위 / 통합 / 시스템 / 인수 테스트)
	2. 애플리케이션 통합 테스트
		1. 목 객체 유형 (더미객체, 테스트 스텁, 테스트 드라이버 등)
	3. 애플리케이션 성능 개선
		1. 애플리케이션 성능 측정 지표 (처리량, 응답 / 경과 시간, 자원 사용률)

13. 제품 소프트웨어 패키징
	1. 제품 소프트웨어 패키징
		1. 릴리즈 노트 작성 항목(헤더, 개요, 목적 등)
